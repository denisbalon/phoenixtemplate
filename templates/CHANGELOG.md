# Changelog

All notable changes per `VERSION` bump. One section per commit on a feature branch. CHANGELOG entries are part of sub-stage (c) of each atomic commit on the on-branch per-node `gogogo!` cadence (alongside the version-marker bump in sub-stage (b)) — they ride in the same commit as the version bump and the code change.

Format: `## v<X.Y.Z> — YYYY-MM-DD` followed by bullets, optionally grouped by area.
