# Project Starter

**Template version:** v1.43.3
**Last updated:** 2026-05-28

The entry-point index for the **phoenixtemplate** kit: a reusable bootstrap for new software projects worked on with Claude Code. The kit's process layer (the `gogogo!` propose-and-confirm gate, on-branch per-node `gogogo!` cadence with N≥1 atomic commits per branch, spec-block format, Karpathy standing rules, reviewer-agnostic PR rubric) is stack-agnostic; the language preset (Python/uv/FastAPI/VPS) is Python-only today, multi-preset is roadmap (D-009 in `docs/spec.md`).

This file is intentionally thin — it points at five companion docs that own the actual content. After v1.26.0's split is complete, each topic lives in its own file:

| Doc | Read when |
|---|---|
| [`BOOTSTRAP.md`](BOOTSTRAP.md) | Starting a new project from the kit — zero to first commit + the pre-feature-code decisions to answer |
| [`WORKFLOW.md`](WORKFLOW.md) | Working on a project (canonical for `gogogo!` gate, on-branch per-node cadence, branching/commits/PR/merge/deploy, conventions, PR review rubric) |
| [`TEMPLATE_INVENTORY.md`](TEMPLATE_INVENTORY.md) | Orienting on the `templates/` directory — file/folder layout + copy-paste reference table |
| [`DEPLOY_BASELINE.md`](DEPLOY_BASELINE.md) | Setting up a VPS deploy — DNS, TLS, reverse proxy, systemd, firewall, CI/CD baseline, credential handling |
| [`HARNESS_QUIRKS.md`](HARNESS_QUIRKS.md) | Claude Code harness gotchas + `bootstrap.sh` design internals |

Plus:

- [`docs/spec.md`](docs/spec.md) — frozen behavior in Block format + Decision log + open project-level decisions for this meta-repo
- [`CHANGELOG.md`](CHANGELOG.md) — per-version diary for this meta-repo
- [`templates/`](templates/) — copy-paste-ready file scaffolds (scripts, doc skeletons, config) that ship with every bootstrapped project

The companion docs ship in the same archive as this file (see `scripts/export-starter.sh` `ROOT_DOCS` array — all six root docs are promoted alongside the flattened `templates/` contents), so cross-links resolve in the consumer's extracted layout.
